How I built my YOLO-based smart traffic system

This project connects real time object detection with physical traffic signals.

I started with the goal of using a camera feed to automatically adjust signal timing based on traffic density. The pipeline was:

  • Collect sample traffic footage from intersections with different densities.
  • Use YOLO to detect vehicles in each frame and count them per lane.
  • Aggregate counts over a short window to estimate density levels.
  • Send commands to an Arduino board which controls LEDs representing signal lights.

The most interesting part was tuning the thresholds so that the signal does not flicker too often, and building a simple protocol between Python and Arduino that is reliable even if frames drop.

Kaggle ¬∑ ML

Horse survival prediction: my winning pipeline

A structured dataset, medical style fields, and a lot of missing data.

The competition was about predicting the survival of horses based on clinical measurements. What worked well:

  • Systematic handling of missing values instead of dropping rows.
  • Target-aware encoding for some categorical fields.
  • Trying multiple models and using an ensemble instead of trusting a single one.
  • Repeated stratified K-fold cross validation to get stable estimates.

The key takeaway for me was that for structured data, feature engineering and validation strategy matter more than chasing the fanciest model.

XR / VR

Building a VR mini game in two days

Notes from the Jio Tesseract XR bootcamp that ended with a first prize.

With limited time, I focused on a simple but polished concept instead of trying to implement complex mechanics. The main pieces were:

  • Designing a small environment that feels coherent and not empty.
  • Making sure core interactions (grab, teleport, trigger) are smooth.
  • Adding small feedback elements like sound and haptics.

This project made me realize that for XR experiences, comfort, clarity and feedback are more important than raw feature count.
